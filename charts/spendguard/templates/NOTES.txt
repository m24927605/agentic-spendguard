SLICE_01 UPGRADE NOTICE — audit chain prediction extension (round-2 fix B2)
================================================================================

If you are upgrading from a pre-SLICE_01 release, you MUST regenerate the
migration ConfigMaps BEFORE running `helm upgrade`. The new release ships
4 additional migration files that the post-install Job will apply:

  Ledger DB:
    - 0046_audit_outbox_prediction_columns.sql   (schema + trigger + TRUNCATE guard)
    - 0048_tokenizer_versions.sql                (FK target + immutability)

  Canonical Ingest DB:
    - 0013_canonical_events_prediction_columns.sql  (mirror schema)
    - 0014_schema_bundle_prediction_v1alpha1.sql    (rotated bundle row)

CRITICAL ORDERING — apply BOTH ledger migrations BEFORE canonical_ingest
migrations. The script in templates/migrations.yaml does this automatically
for ConfigMap-mode installs.

REGENERATE the ConfigMaps:
  scripts/helm-validate-kind.sh                 # regenerates from on-disk files
  # OR manually:
  kubectl --namespace {{ .Release.Namespace }} create configmap \
    {{ .Values.migrations.ledgerConfigMapName | default "spendguard-migrations-ledger" }} \
    --from-file=services/ledger/migrations/ \
    --dry-run=client -o yaml | kubectl apply -f -
  kubectl --namespace {{ .Release.Namespace }} create configmap \
    {{ .Values.migrations.canonicalConfigMapName | default "spendguard-migrations-canonical" }} \
    --from-file=services/canonical_ingest/migrations/ \
    --dry-run=client -o yaml | kubectl apply -f -
